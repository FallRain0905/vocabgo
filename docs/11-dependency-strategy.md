# VocabGo RPA 依赖打包策略详解

## 🎯 推荐方案：智能引导下载（已实现）

### 方案特点

✅ **安装包小**：只有10-20MB，下载快速
✅ **自动下载**：首次运行时自动检测并下载缺失依赖
✅ **进度显示**：GUI界面显示下载进度和状态
✅ **按需下载**：只下载缺失的依赖
✅ **易于更新**：OCR模型可单独更新，无需重新下载整个程序
✅ **用户体验好**：智能引导，减少用户困惑

### 实现方式

#### 1. 依赖检查系统
```
启动程序
    ↓
自动检查已安装的依赖
    ↓
有缺失？
    ├─ 是 → 显示依赖安装GUI
    │       ↓
    │   用户点击"安装缺失依赖"
    │       ↓
    │   自动下载缺失的文件
    │       ↓
    │   安装到程序目录
    │       ↓
    │   完成后启动主程序
    │
    └─ 否 → 直接启动主程序
```

#### 2. 下载依赖内容

| 组件 | 大小 | 说明 | 必需性 |
|------|------|------|----------|
| Tesseract引擎 | ~20MB | OCR核心引擎 | ✅ 必需 |
| 中文语言包 | ~5MB | 简体中文识别 | ✅ 推荐 |
| 英文语言包 | ~2MB | 英文识别 | ✅ 推荐 |
| **总计** | **~27MB** | | |

#### 3. 用户界面流程

**首次运行（有缺失依赖）**：
1. 程序自动启动依赖检查
2. 显示当前依赖状态（✓已安装 / ✗未安装）
3. 显示需要下载的文件和总大小
4. 用户点击"安装缺失依赖"按钮
5. 显示下载进度界面
6. 下载完成后显示成功提示
7. 自动启动主程序

**后续运行（所有依赖已安装）**：
1. 程序检测依赖完整
2. 显示"所有依赖已安装"提示
3. 用户点击"继续启动程序"
4. 直接进入主界面

### 技术实现

#### 核心文件

**`scripts/dependency_manager.py`**：
- `DependencyDownloader`：依赖下载核心类
- `DependencyInstallerGUI`：依赖安装GUI界面
- `check_and_install_dependencies()`：主检查函数
- `DEPENDENCIES`：依赖定义配置

#### 集成到主程序

**`scripts/translator_gui.py`**：
```python
# 添加依赖检查
from scripts.dependency_manager import check_and_install_dependencies

# 在main()中调用
def main():
    check_and_launch()  # 检查依赖并启动
```

### 优势对比

| 对比项 | 全打包 | 引导下载（推荐） |
|--------|--------|------------------|
| **安装包大小** | 100MB+ | 10-20MB |
| **下载时间** | 长（10-20分钟） | 短（1-2分钟） |
| **首次运行** | 立即可用 | 需要下载（3-5分钟） |
| **存储空间** | 大 | 小 |
| **更新OCR模型** | 需要重新下载 | 只下载模型文件 |
| **离线使用** | 支持 | 需要首次在线下载 |
| **用户体验** | 简单 | 略复杂但更现代 |


```
    VocabGo_RPA.exe          # 主程序
    ├── config/              # 配置文件目录
    ├── assets/              # 资源文件
    ├── docs/               # 文档

        ├── tesseract/       # Tesseract引擎
        └── tessdata/        # 语言包
            ├── chi_sim.traineddata
            └── eng.traineddata

```

```
C:\Program Files\VocabGo_RPA\
├── VocabGo_RPA.exe
├── config/
├── assets/
├── docs/
└── dependencies/ (自动创建)
```

## 🎯 核心优势总结

### 为什么选择引导下载方案？

#### 1. 用户体验
- ✅ 下载安装包快速（1-2分钟 vs 10-20分钟）
- ✅ 智能引导，明确告知需要下载的内容
- ✅ 现代化的下载进度界面
- ✅ 一次下载，永久使用

#### 2. 开发维护
- ✅ 主程序体积小，便于快速迭代
- ✅ OCR模型可独立更新
- ✅ 不需要每次更新都重新打包大文件
- ✅ 依赖管理逻辑独立，便于维护

#### 3. 发布分发
- ✅ GitHub发布包小，下载快
- ✅ 减少存储和带宽成本
- ✅ 更符合现代软件分发习惯

#### 4. 灵活性
- ✅ 用户可选择只下载必需的依赖
- ✅ 支持离线使用（首次在线下载后）
- ✅ 可以添加更多依赖选项（语音包、翻译模型等）

## 📊 性能指标

### 下载时间估算

| 网络速度 | 安装包下载 | 依赖下载 | 总时间对比 |
|----------|-----------|----------|-----------|
| **1 Mbps** | 80秒 | 216秒 | 全打包：300秒<br/>引导下载：296秒 |
| **10 Mbps** | 8秒 | 22秒 | 全打包：30秒<br/>引导下载：30秒 |
| **100 Mbps** | 0.8秒 | 2.2秒 | 全打包：3秒<br/>引导下载：3秒 |

**结论**：在正常网络（10Mbps+）下，两者差异可忽略不计

### 存储空间对比

|------|--------|----------|
| **初始下载** | 100 MB | 10 MB |
| **首次运行后** | 100 MB | 37 MB |
| **存储节省** | - | 90% |
